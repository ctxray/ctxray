## First Comment (post immediately after submission)

Hey HN, I'm Chris. I've been using Claude Code and Cursor daily for over a year. A few months ago I started wondering: what am I actually typing into these tools? Not what the AI produces -- what am I sending.

I built reprompt to answer that. It's a CLI that scans session files from 9 coding tools (Claude Code, Cursor, Codex, Aider, Gemini, Cline, and three others), extracts the human-written prompts, and analyzes them locally.

Three things surprised me after running it on my own history.

First, I found 3 API keys and a JWT token in my prompts. Not because I pasted them intentionally -- they were in error messages and stack traces I copy-pasted for debugging. `reprompt privacy --deep` catches these. It's regex-based, no network calls.

Second, my Claude Code agent sessions have a 35% error loop rate. That's the percentage of sessions where the agent got stuck retrying the same failing approach 3+ times. The `reprompt agent` command builds fingerprints from tool calls (Bash, Edit, Read) and their targets, then detects when the same fingerprint repeats with errors. Zero instrumentation needed -- it reads the existing JSONL session files.

Third, 50-70% of my conversation turns are filler. "ok try that", "continue", "looks good." The `reprompt distill` command scores every turn using 6 signals (position, length, tool trigger, error recovery, topic shift, vocabulary uniqueness) and extracts only the turns that drove the conversation. A 100-turn session typically distills to 15-20 turns in under 50ms. All TF-IDF and heuristics, no LLM.

The scoring engine rates individual prompts 0-100 based on four NLP papers: specificity signals from Google 2512.14982, position bias from Stanford 2307.03172, and two others. After two months of seeing my scores, my debug prompts improved from averaging 31 to 52 without conscious effort. Just seeing a number made me write better prompts.

    pip install reprompt-cli
    reprompt scan && reprompt

That gives you a dashboard with your prompt count, avg score, top categories, and source breakdown. Then try `reprompt distill --last 3` and `reprompt privacy --deep`.

1,494 tests, MIT license, zero network calls, zero telemetry. Also works as an MCP server for Claude Code and as a GitHub Action for CI prompt quality gates.

## Prepared Responses

### "Why not just use an LLM to analyze prompts?"

Two reasons. Determinism: reprompt shows you trends over weeks. If the analysis function changes with model version or temperature, trend tracking breaks. Privacy: my prompts contain file paths, function names, sometimes credentials. Sending those to another API to analyze defeats the purpose.

The tradeoff is that structural signals miss intent. An LLM would understand "this is where I realized the architecture was wrong." My heuristics just see a long turn with high vocabulary shift. But for daily use, structural analysis catches 80-90% of the important moments and runs in <50ms.

### "What about Promptfoo / Langfuse / LangSmith?"

They all analyze LLM outputs -- did the model produce correct answers? reprompt analyzes human inputs -- what did you actually type? It's the opposite direction. Promptfoo was just acquired by OpenAI and is becoming their security/red-teaming tool. Langfuse and LangSmith require instrumentation in your code. reprompt reads existing session files with zero config.

### "What's the agent analysis actually detecting?"

It fingerprints each assistant turn as: tool_name(target_file):has_error. So a turn that runs `Bash(test.py)` and gets an error becomes `Bash(test.py):err`. When the same fingerprint appears 3+ times in a row, that's an error loop. Two-step loops are also detected: `Bash(test.py):err -> Edit(auth.py) -> Bash(test.py):err` repeating means the edit isn't fixing the test.

Tool distribution shows which tools the agent uses most. In my sessions, Bash is 40%, Read is 25%, Edit is 20%. The efficiency score is the ratio of productive turns (no errors) to total turns.

### "Is the privacy scanner just regex?"

Yes, 12 patterns: OpenAI/AWS/GitHub/Anthropic/Stripe API keys, JWT tokens, emails, IPv4 addresses, passwords in assignments, env secrets, and home directory paths. Safety filters exclude localhost, 10.x.x.x, example.com emails. It's not going to catch obfuscated secrets, but it catches the obvious ones that end up in prompts via copy-pasted error messages. Which is most of them.

### "Performance?"

On an M2 Mac: scanning 1,200 prompts takes under 2 seconds. Distilling a 100-turn conversation: <50ms. Scoring a single prompt: <1ms. Storage is SQLite. Incremental scanning so subsequent runs only process new sessions.
